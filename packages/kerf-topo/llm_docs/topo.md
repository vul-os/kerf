# kerf-topo — topology optimisation (UFL-SIMP)

`kerf-topo` implements density-based topology optimisation using the SIMP (Solid Isotropic Material with Penalization) method. It is designed for structural weight-minimisation problems: given a design domain, loads, and boundary conditions, it finds the material distribution that minimises compliance (maximises stiffness) for a given volume fraction target.

Heavy deps (`dolfinx`, `gmsh`, `scikit-image`, `scipy`) are optional. The plugin always loads but declares `topo.simp` only when both `dolfinx` and `scikit-image` are available.

---

## Plugin registration

```python
async def register(app, ctx):
    app.include_router(router)                    # POST /run-topo
    ctx.tools.register("topo_run", topo_run_spec, run_topo_run)
    provides = ["topo.simp"] if dolfinx + skimage available else []
    return PluginManifest(
        name="topo",
        provides=provides,
        depends=["fem", "cad-core"],
    )
```

## LLM tool: `topo_run`

Queues a topology optimisation job. Returns `job_id` for polling via the standard job-status pattern.

```json
{
  "file_id": "<uuid of the FEM/domain setup file>",
  "volume_fraction": 0.4,
  "penalty": 3.0,
  "filter_radius": 1.5,
  "max_iterations": 80,
  "min_change": 0.01
}
```

Parameters:
- `volume_fraction` — target fraction of domain to retain (0.0–1.0; default 0.4)
- `penalty` — SIMP penalization exponent (default 3.0; higher = sharper 0/1 boundaries)
- `filter_radius` — mesh element filter radius in mm (prevents checkerboard patterns)
- `max_iterations` — iteration limit (default 80)
- `min_change` — convergence threshold on density change between iterations (default 0.01)

Returns when done:
```json
{
  "job_id": "<uuid>",
  "status": "done",
  "result": {
    "density_field_b64": "<base64 numpy array>",
    "compliance_history": [12.3, 8.7, 6.1, ...],
    "volume_fraction_achieved": 0.401,
    "iterations": 54,
    "converged": true,
    "mesh_file_id": "<uuid of output ISO-surface mesh>"
  }
}
```

The output ISO-surface mesh (marching cubes at `density ≥ 0.5`) is stored as a `kind='mesh'` file in the project and can be used for downstream FEM validation, export to STEP, or rendering.

## `/run-topo` route

Synchronous endpoint used by the pyworker. Accepts the full problem specification (mesh, boundary conditions, loads) and runs the SIMP loop. Returns the density field and convergence data.

The route is not typically called directly — the `topo_run` LLM tool dispatches to the pyworker, which calls this endpoint.

---

## SIMP algorithm (`kerf_topo.tools`, `kerf_topo.advanced`)

The implementation:

1. **Mesh** — provided by the FEM file or generated by `gmsh` from the design domain STEP
2. **FE solve** — linear elasticity in FEniCS/dolfinx; displacement field U
3. **Sensitivity** — compliance sensitivity `∂C/∂ρ = -p·ρ^(p-1)·Ue^T·K0·Ue`
4. **Filter** — density filter with radius r (PDE filter or explicit convolution)
5. **OC update** — optimality criteria update on element densities
6. **Convergence** — check `max|Δρ| < min_change` or `iterations == max_iterations`
7. **ISO surface** — `skimage.measure.marching_cubes` at threshold 0.5
8. **Export** — mesh as `kind='mesh'` file; density array as derived artifact

The `advanced.py` module adds:
- Stress-constrained topology optimisation (P-norm stress aggregation)
- Multi-load case compliance minimisation (sum of compliances)
- Thermal topology optimisation (heat conduction minimisation)

## Model selection note

The `/run-topo` agent is intended to run with `claude-opus-4-7` for full UFL problem formulation capability (FEniCS UFL expressions, boundary condition reasoning, mesh generation). Using Sonnet degrades the quality of constraint interpretation for complex multi-load cases. See `opus_for_topology_optimization` in the project memory.

## Dependencies

| Package | Role |
|---|---|
| `dolfinx` | FEniCS-X FEM solver (linear elasticity, UFL forms) |
| `gmsh` | Mesh generation from STEP boundary |
| `scikit-image` | Marching cubes for ISO-surface extraction |
| `scipy` | Sparse linear algebra, optimality criteria solver |
| `pythonocc-core` | STEP import for the design domain |

All heavy deps run in the pyworker sidecar, not in the main FastAPI process.
