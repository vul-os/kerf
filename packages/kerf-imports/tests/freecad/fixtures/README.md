# FreeCAD import test fixtures

These `.FCStd` files are minimal valid archives for use in the `kerf-imports`
test suite. They were generated by `scripts/generate_freecad_fixtures.py`
using the pure-Python `zipfile` + `xml` builder (no `freecadcmd` required).

## Fixtures

| File | Description |
|------|-------------|
| `single_pad.FCStd` | One Body, one Sketch (rectangle, 4 lines), one Pad (10mm). |
| `pad_and_pocket.FCStd` | One Body, Sketch + Pad + SketchPocket + Pocket. |
| `two_bodies.FCStd` | Two separate `PartDesign::Body` objects → exercises assembly creation. |
| `sketch_constraints.FCStd` | Sketch with Coincident, Distance, Angle, Tangent, Radius constraints. |
| `unsupported_constraints.FCStd` | Sketch with SnellsLaw (type 16), Weight (type 19), InternalAlignment (type 15) → all dropped with warnings. |

## Regenerating

These fixtures contain no real BRep geometry (the `.brp` members are absent).
They are sufficient for testing the parser, sketch translator, feature metadata
capture, assembly builder, and route — all of which operate on `Document.xml`
content.

To regenerate with the minimal builder:

```bash
python scripts/generate_freecad_fixtures.py \
  --output-dir packages/kerf-imports/tests/freecad/fixtures
```

To regenerate with real BRep geometry (requires FreeCAD ≥ 0.19):

```bash
freecadcmd scripts/generate_freecad_fixtures.py \
  --output-dir packages/kerf-imports/tests/freecad/fixtures
```

Commit the resulting `.FCStd` files. CI does not regenerate them.
