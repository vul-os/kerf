The Workshop isn't a server.
It's a protocol.

Sharing a part shouldn't require an account on someone else's platform. Kerf's Workshop is built on DMTAP-PUB, an open protocol for publishing signed, content-addressed objects — the same idea behind Nostr's feeds and IPFS's content addressing, purpose-built for engineering artifacts.

  • ✓No accounts. Your identity is a keypair you hold, not a login on a database you don't control.
  • ✓No central server. Any gateway — yours, a friend's, kerf.sh's — can serve any object, because every object proves itself.
  • ✓A workshop is just feeds you follow. Publishing appends to your own feed; browsing means fetching feeds you chose.
  • ✓Offline once pinned. Content-addressed and self-verifying — a part you've pinned works with zero live infrastructure.
  • ✓Plain HTTPS today. No mesh client required to publish or follow — a handful of GET endpoints is a complete implementation.

download

Self-host in minutes

No account, no cloud, no external service beyond a Postgres database you already control.

install
$ curl -fsSL https://kerf.sh/install.sh | sh

Downloads the latest GitHub release, unpacks it, and sets up a Python venv. Then open http://localhost:8080. Prefer a specific asset? Browse all releases — every tag ships kerf-vX.Y.Z-macos-arm64.tar.gz, -macos-x64.tar.gz, -linux-x64.tar.gz, a universal -src.tar.gz, and a SHA256SUMS manifest.

or, docker
$ git clone https://github.com/kerf-sh/kerf $ cd kerf $ docker compose up # app + postgres + redis, builds the full persona

Prefer running from source? See docs/getting-started.md for the pip + npm walkthrough.
